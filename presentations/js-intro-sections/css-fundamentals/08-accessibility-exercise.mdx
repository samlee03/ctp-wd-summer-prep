describe('Accessibility Fixes', () => {
    it('should have proper heading hierarchy', () => {
        const h1 = document.querySelector('h1');
        const h2s = document.querySelectorAll('h2');
        expect(h1).to.exist;
        expect(h2s.length).to.be.at.least(2);
    });
    
    it('should have alt text for images', () => {
        const images = document.querySelectorAll('img');
        images.forEach(img => {
            expect(img.getAttribute('alt')).to.not.be.empty;
        });
    });
    
    it('should have labels for form inputs', () => {
        const inputs = document.querySelectorAll('input, select');
        inputs.forEach(input => {
            const label = document.querySelector(`label[for="${input.id}"]`);
            expect(label).to.exist;
        });
    });
    
    it('should have sufficient color contrast', () => {
        const header = document.querySelector('.site-header');
        const styles = window.getComputedStyle(header);
        // Check that dark background has light text
        expect(styles.color).to.equal('rgb(255, 255, 255)');
    });
    
    it('should have keyboard support for interactive elements', () => {
        const card = document.querySelector('.habit-card');
        expect(card.getAttribute('tabindex')).to.equal('0');
        expect(card.getAttribute('onkeypress')).to.exist;
    });
    
    it('should have skip navigation link', () => {
        const skipLink = document.querySelector('.skip-link');
        expect(skipLink).to.exist;
        expect(skipLink.getAttribute('href')).to.equal('#main-content');
    });
});

<Notes>
This exercise challenges students to identify and fix common accessibility issues. Each problem represents a real-world mistake that makes websites unusable for many people. Let's examine what needs fixing and why.

The color contrast issue (#555 text on #333 background) fails WCAG standards badly. Students need to increase contrast to at least 4.5:1. The lack of semantic HTML (using divs instead of header, h1, button elements) means screen readers can't understand the page structure. The onclick handler on a div makes it inaccessible to keyboard users.

The missing alt text on the image leaves screen reader users wondering what the image shows. The color-only error indicator excludes colorblind users. The form without labels is unusable for screen readers - they won't know what each input is for.

This exercise teaches that accessibility isn't one thing - it's many small decisions that add up. Each fix makes the site usable for more people. Students learn to think about different ways people interact with websites and build empathy for users with different abilities. The goal is to make accessibility checking a natural part of their development process.
